# Runbook: Domain Change

How to change the platform's configured domain. This affects all environments.

---

## When this is needed

- You're moving from a placeholder/test domain to a real domain
- A customer is using their own domain
- You're transferring to a different registrar or TLD

---

## Step 1: Register/configure the new domain

In your DNS registrar:

- Create an A record: `<new-domain>` → `<SERVER_IP>`
- Create a wildcard A record: `*.<new-domain>` → `<SERVER_IP>`

Wait for DNS to propagate:

```bash
nslookup dev.<new-domain> 8.8.8.8
```

---

## Step 2: Update Coolify environment variables

For each active environment (dev, and staging/prod if active):

1. In Coolify, open the resource settings
2. Update `DOMAIN` to the new domain
3. Update `NEXT_PUBLIC_APP_URL` to `https://dev.<new-domain>` (or staging/prod equivalent)
4. Update `ADMIN_EMAIL` if the admin contact email is domain-specific

---

## Step 3: Update the Caddyfile

In `infra/caddy/Caddyfile.platform`, the domain is read from `{$DOMAIN}` — no change needed to the Caddyfile itself, as Coolify injects the `DOMAIN` env var into the Caddy context.

If you have hardcoded domain references in Coolify's Caddy config (generated by Coolify), update them.

---

## Step 4: Trigger redeployment

Trigger a redeploy for each active environment via GitHub Actions or directly in Coolify.

Caddy will automatically obtain new SSL certificates for the new domain from Let's Encrypt.

---

## Step 5: Verify SSL

```bash
curl -I https://dev.<new-domain>
# Should return HTTP/2 200 with a valid Let's Encrypt certificate
```

SSL Labs test: `https://www.ssllabs.com/ssltest/analyze.html?d=dev.<new-domain>`

---

## Step 6: Old domain cleanup

- Keep old DNS records pointing to the server for 30 days (to catch any cached references)
- After 30 days, remove the old DNS records
- If using the old domain for email (e.g., `ADMIN_EMAIL`), ensure email still works

---

## Step 7: Update local development

Update `.env.local`:

```
DOMAIN=dev.<new-domain>
NEXT_PUBLIC_APP_URL=https://dev.<new-domain>
```

Notify other developers to update their local env files.
