# Contract: Persistence Ports

**Package:** `@platform/ports-persistence`

## Purpose

Provides the full suite of persistence abstractions used by platform core. Adapters implement these ports
for PostgreSQL, MSSQL, and MongoDB. Core never imports adapter packages; all persistence access goes
through these ports.

Six ports compose the surface:

| Port                      | Responsibility                                               |
| ------------------------- | ------------------------------------------------------------ |
| `RepositoryPort<TEntity>` | CRUD, filtering, pagination, soft delete, optimistic locking |
| `UnitOfWorkPort`          | Transaction boundary                                         |
| `SchemaIntrospectionPort` | Read live schema metadata from the database                  |
| `SchemaDdlPort`           | Generate and apply DDL from normalised `TableDefinition`     |
| `SchemaMigrationPort`     | Apply and revert versioned migrations                        |
| `QueryPort`               | Typed, read-only complex queries                             |

## Key Types

```typescript
// Filter AST — compose with _and / _or / _not
type Filter<T> = {
  _and?: Filter<T>[];
  _or?: Filter<T>[];
  _not?: Filter<T>;
  _eq?: Partial<T>;
  _neq?: Partial<T>;
  _in?: { [K in keyof T]?: T[K][] };
  _nin?: { [K in keyof T]?: T[K][] };
  _lt?: Partial<T>;
  _lte?: Partial<T>;
  _gt?: Partial<T>;
  _gte?: Partial<T>;
  _contains?: { [K in keyof T]?: string }; // case-sensitive substring
  _icontains?: { [K in keyof T]?: string }; // case-insensitive substring
  _starts_with?: { [K in keyof T]?: string };
  _ends_with?: { [K in keyof T]?: string };
  _is_null?: { [K in keyof T]?: boolean };
};

type Sort<T> = { field: keyof T; direction: 'asc' | 'desc' }[];

type Page = { limit: number; offset: number };

type PaginatedResult<T> = { items: T[]; total: number; limit: number; offset: number };

type PlatformColumnType = 'string' | 'text' | 'integer' | 'bigint' | 'decimal' | 'boolean' | 'date' | 'timestamp' | 'timestamp_tz' | 'uuid' | 'binary' | 'json' | 'array';

type SchemaFeature = 'schemas' | 'foreign_keys' | 'check_constraints' | 'computed_columns' | 'json_columns' | 'array_columns' | 'partial_indexes' | 'unique_indexes' | 'spatial_indexes' | 'transactions' | 'change_streams';
```

## Methods

### RepositoryPort\<TEntity\>

#### `findById(id: string, ctx: RequestContext): Promise<Result<TEntity | null, PersistenceError>>`

**Pre-conditions:** `id` is a non-empty string; `ctx.workspaceId` is set.
**Post-conditions:** Returns the entity scoped to the caller's workspace, or `null` if not found. Never
returns an entity belonging to a different workspace.
**Invariants:** The workspace scope check is applied at the query level, not in application code.

#### `findMany(filter: Filter<TEntity>, sort: Sort<TEntity>, page: Page, ctx: RequestContext): Promise<Result<PaginatedResult<TEntity>, PersistenceError>>`

**Pre-conditions:** `page.limit` is between 1 and 1000 (inclusive). `page.offset` is non-negative.
**Post-conditions:** `total` reflects the count of matching rows before pagination; `items.length <=
page.limit`.
**Invariants:** All items belong to `ctx.workspaceId`.

#### `create(entity: Omit<TEntity, 'id' | 'version' | 'createdAt' | 'updatedAt'>, ctx: RequestContext): Promise<Result<TEntity, PersistenceError>>`

**Pre-conditions:** Caller must have authorised the write before calling this method. Required unique
constraints are not violated.
**Post-conditions:** Returns the persisted entity with `id`, `version = 1`, `createdAt`, and `updatedAt`
populated.
**Invariants:** `id` is generated by the adapter (UUID or equivalent). Version is always 1 on first
insert.

#### `update(input: { id: string; expectedVersion: number; changes: Partial<TEntity> }, ctx: RequestContext): Promise<Result<TEntity, PersistenceError | ConflictError>>`

**Pre-conditions:** `expectedVersion` matches the current `version` column in the database.
**Post-conditions:** `version` is incremented by 1; `updatedAt` is updated.
**Invariants:** If `expectedVersion` does not match, returns `ConflictError` — never silently overwrites.

#### `archive(id: string, expectedVersion: number, ctx: RequestContext): Promise<Result<void, PersistenceError | ConflictError>>`

Soft delete. Sets `archivedAt` to the current timestamp; entity remains queryable via explicit filter
`{ _is_null: { archivedAt: false } }`. Standard `findMany` excludes archived rows unless the filter
explicitly includes them.

**Pre-conditions:** `expectedVersion` matches current version.
**Post-conditions:** Entity is soft-deleted. Subsequent unfiltered `findMany` omits it.

#### `hardDelete(id: string, expectedVersion: number, ctx: RequestContext): Promise<Result<void, PersistenceError | ConflictError>>`

Permanently removes the row. Irreversible. Use only when data retention policy requires it.

**Pre-conditions:** Caller holds the `hard_delete` permission for this resource type. `expectedVersion`
matches.
**Post-conditions:** Row is gone. Any foreign-key references become dangling (callers must clean up
first).

### UnitOfWorkPort

#### `run<T>(work: (uow: UnitOfWork) => Promise<T>, ctx: RequestContext): Promise<Result<T, PersistenceError>>`

Executes `work` inside a single database transaction. If `work` throws or returns an error, the
transaction is rolled back automatically.

**Pre-conditions:** The adapter supports `transactions` (check `SchemaIntrospectionPort.supports`
before calling).
**Post-conditions:** Either all writes inside `work` commit atomically, or none do.
**Invariants:** The `uow` handle passed to `work` must be used for all repository calls that should
participate in the transaction — calls made with a separate repository instance are not included.
Never nest `run` calls; the adapter must detect nesting and return a `PersistenceError`.

### SchemaIntrospectionPort

#### `listSchemas(ctx: RequestContext): Promise<Result<string[], PersistenceError>>`

Returns schema/namespace names. On MongoDB this returns an empty array; check
`supports('schemas')` first.

#### `listTables(schema: string | null, ctx: RequestContext): Promise<Result<TableSummary[], PersistenceError>>`

`schema` is `null` for databases that have no schema namespace (MongoDB, some MSSQL configurations).

#### `describeTable(table: string, schema: string | null, ctx: RequestContext): Promise<Result<TableDefinition, PersistenceError>>`

Returns the normalised column list, using `PlatformColumnType` for each column. Adapter-specific
types that have no platform equivalent are mapped to `'string'` with an `adapterType` hint on the
column descriptor.

#### `listIndexes(table: string, schema: string | null, ctx: RequestContext): Promise<Result<IndexDefinition[], PersistenceError>>`

#### `listForeignKeys(table: string, schema: string | null, ctx: RequestContext): Promise<Result<ForeignKeyDefinition[], PersistenceError>>`

Returns an empty array on MongoDB (no real foreign keys).

#### `listConstraints(table: string, schema: string | null, ctx: RequestContext): Promise<Result<ConstraintDefinition[], PersistenceError>>`

#### `supports(feature: SchemaFeature): boolean`

Synchronous capability check. Call this before any feature-gated operation. Adapters must not throw
`NotSupportedError` for a feature that `supports()` returns `true` for — the two must stay in sync.

---

### SchemaDdlPort

Generates and applies DDL from a `TableDefinition`. Adapters translate `PlatformColumnType` to the
appropriate native type.

#### `createTable(definition: TableDefinition, ctx: RequestContext): Promise<Result<void, PersistenceError>>`

**Pre-conditions:** Table does not already exist. All `PlatformColumnType` values in the definition
are supported by this adapter (check `supports()` for `array_columns`, `json_columns`, etc.).
**Post-conditions:** Table exists with the described columns, indexes, and constraints.

#### `alterTable(table: string, schema: string | null, changes: TableAlteration, ctx: RequestContext): Promise<Result<void, PersistenceError>>`

`TableAlteration` describes `addColumns`, `dropColumns`, `alterColumns`, `addConstraints`,
`dropConstraints`. Adapters may reject destructive alterations when rows exist; callers should
check `supports('check_constraints')` before relying on constraint additions.

#### `dropTable(table: string, schema: string | null, opts: { cascade?: boolean }, ctx: RequestContext): Promise<Result<void, PersistenceError>>`

**Pre-conditions:** Caller has confirmed intent (this is irreversible). `cascade: true` is required
if foreign keys reference this table; not supported on MongoDB.

#### `createIndex(definition: IndexDefinition, ctx: RequestContext): Promise<Result<void, PersistenceError>>`

**Pre-conditions:** `supports('partial_indexes')` is `true` when `definition.where` is set.

#### `validate(definition: TableDefinition): Result<void, ValidationError[]>`

Synchronous. Validates the definition against this adapter's capabilities before sending any DDL.
Returns all violations, not just the first. Callers should run this before `createTable` or
`alterTable` to surface problems early.

---

### SchemaMigrationPort

#### `apply(migration: Migration, ctx: RequestContext): Promise<Result<void, PersistenceError>>`

Applies a single migration and records it in the migrations ledger table.

**Invariants:** If a migration with the same version is already applied, returns a `ConflictError`
rather than applying again.

#### `revert(version: string, ctx: RequestContext): Promise<Result<void, PersistenceError>>`

Calls the migration's `down` function and removes it from the ledger.

**Pre-conditions:** The migration is the most recently applied one (adapters must reject out-of-order
reverts).

#### `listApplied(ctx: RequestContext): Promise<Result<MigrationRecord[], PersistenceError>>`

#### `listPending(migrations: Migration[], ctx: RequestContext): Promise<Result<Migration[], PersistenceError>>`

Returns migrations in `migrations` that have no corresponding record in the ledger.

---

### QueryPort

Read-only. For complex queries that exceed what `RepositoryPort.findMany` can express (joins,
aggregates, projections across multiple collections/tables).

#### `execute<TResult>(query: TypedQuery<TResult>, ctx: RequestContext): Promise<Result<TResult, PersistenceError>>`

`TypedQuery` is a builder-pattern object produced by the query DSL. The adapter compiles it to
SQL or an aggregation pipeline. Workspace scoping must be embedded in the query; `QueryPort` does
not inject it automatically — callers are responsible.

## Capability Flags

| Feature             | Postgres                  | MSSQL     | MongoDB                           |
| ------------------- | ------------------------- | --------- | --------------------------------- |
| `schemas`           | yes                       | yes       | no                                |
| `foreign_keys`      | yes                       | yes       | no (advisory only)                |
| `check_constraints` | yes                       | yes       | no                                |
| `computed_columns`  | yes                       | yes       | no                                |
| `json_columns`      | yes                       | partial   | yes (native BSON)                 |
| `array_columns`     | yes                       | no        | yes (native arrays)               |
| `partial_indexes`   | yes                       | no        | yes (sparse indexes)              |
| `unique_indexes`    | yes                       | yes       | yes                               |
| `spatial_indexes`   | yes (PostGIS)             | yes       | yes (2dsphere)                    |
| `transactions`      | yes                       | yes       | no (without replica set sessions) |
| `change_streams`    | yes (logical replication) | yes (CDC) | yes                               |

## Performance Expectations

- `findById`: < 5 ms on a warm connection with a primary-key index.
- `findMany` with a selective filter and index: < 20 ms for up to 10,000 matching rows.
- `findMany` without an index on the filter field: performance degrades linearly — adapters must
  not add implicit full-scans without logging a warning.
- `UnitOfWorkPort.run`: keep transactions short; the platform enforces a 30-second timeout on the
  callback before rolling back.
- `QueryPort.execute`: no implicit timeout; callers are expected to set one via `ctx.deadline`.

---

## Known Adapter Divergences

**PostgreSQL**
Full support for all features. The reference implementation. Use as the baseline when writing
conformance tests.

**MSSQL**

- `array_columns` are not supported; use `json_columns` or a child table.
- `uuid` maps to `uniqueidentifier`; binary encoding differs from Postgres — do not assume byte order.
- `partial_indexes` (`filtered indexes` in MSSQL syntax) are not surfaced as `supports('partial_indexes') = true`; the adapter does not yet implement them.
- `transactions` supported via `BEGIN TRANSACTION`; distributed transactions are out of scope.

**MongoDB**

- `SchemaDdlPort` is a no-op adapter. `createTable` creates a collection with no schema enforcement.
  `alterTable`, `dropTable`, `createIndex` work as expected; DDL validation is minimal.
- `SchemaIntrospectionPort.listForeignKeys` always returns `[]`.
- `UnitOfWorkPort` requires a replica set; the adapter checks at boot and sets `supports('transactions') = false` on standalone instances.
- `Filter._icontains` compiles to a case-insensitive regex; performance degrades on large collections without a text index.

## Usage Examples

```typescript
// Workspace-scoped paginated list with filtering
const result = await userRepo.findMany({ _and: [{ _eq: { status: 'active' } }, { _is_null: { archivedAt: true } }] }, [{ field: 'createdAt', direction: 'desc' }], { limit: 20, offset: 0 }, ctx);

// Optimistic update — ConflictError when version is stale
const updateResult = await userRepo.update({ id: user.id, expectedVersion: user.version, changes: { displayName: 'New Name' } }, ctx);
if (updateResult.isErr() && updateResult.error.kind === 'version_mismatch') {
  return err(new ConflictError('User was modified concurrently'));
}

// Transaction spanning two repositories
const txResult = await uow.run(async (handle) => {
  await handle.orders.create(order, ctx);
  await handle.inventory.update({ id: item.id, expectedVersion: item.version, changes: { stock: item.stock - 1 } }, ctx);
}, ctx);

// Capability-gated DDL
if (ddl.supports('array_columns')) {
  definition.columns.push({ name: 'tags', type: 'array', itemType: 'string' });
}
const errors = ddl.validate(definition);
if (errors.isErr()) throw new Error(errors.error.map((e) => e.message).join(', '));
await ddl.createTable(definition, ctx);
```

## Common Misuse

- **Forgetting workspace scoping in `QueryPort`.** `RepositoryPort` injects workspace scope
  automatically; `QueryPort` does not. Every `TypedQuery` must include a workspace predicate.
- **Calling `hardDelete` without cleaning foreign keys first.** On relational adapters this fails
  with `CONSTRAINT_VIOLATION`. On MongoDB it silently leaves dangling references.
- **Ignoring `ConflictError` from `update`.** Swallowing the error causes silent last-write-wins
  semantics, defeating optimistic locking.
- **Nesting `UnitOfWorkPort.run` calls.** The outer transaction is not promoted to a savepoint.
  The adapter returns a `PersistenceError` with code `UNKNOWN` — easy to miss. Design for flat
  transaction boundaries.
- **Using `Filter._icontains` on unindexed large collections in MongoDB.** This compiles to a
  full regex scan. Require a text index or restrict the query to indexed fields.
- **Calling `SchemaDdlPort` methods on MongoDB without checking `supports('foreign_keys')`.** The
  DDL adapter accepts the call and ignores the foreign-key definition silently; the Schema Designer
  UI must surface this gap.
