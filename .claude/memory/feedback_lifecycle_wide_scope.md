When discussing the platform's scope — especially for auto-evolution, integration opportunities, idea-scouting, or "what could fit" questions — frame across the **entire lifecycle**, not a single stage. The platform covers: ideation/PRD → documentation → schema design → code generation → testing → deployment → observability → maintenance → support.

**Why:** User corrected a narrow framing where I treated "deployment integrations" as the example category. The platform is a full build-to-maintain pipeline (per master-plan.md and the 30 objectives), so any adjacency map, watch list, or scope discussion must cover all stages. Narrowing to one stage misses most of the surface.

**How to apply:** When asked about integrations, opportunities, or external tools that "fit," produce a lifecycle-wide breakdown by default. Only narrow to a single stage when the user explicitly scopes the question there.
