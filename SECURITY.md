# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, email: **theuns@barnardt.co.za** with:

- A description of the vulnerability
- Steps to reproduce
- The potential impact
- Any suggested fix (optional)

**Disclosure timeline:**

1. You report the vulnerability
2. We acknowledge within 5 business days
3. We investigate and develop a fix
4. We release the fix and publicly disclose within 90 days of the report

We will credit reporters unless anonymity is requested.

---

## Supported Versions

The platform is pre-release (foundation phase). Only the latest code on `main` is supported.
